Linux Commands & How to Login to a Docker Container
Run your own Docker Image and Login to your Docker Container:

Follow along with the steps from the build and run your own image webpage below.

Step 1: Clone the given git repo with "git clone https://github.com/dockersamples/node-bulletin-board" then go into the directory with "cd node-bulletin-board/bulletin-board-app"

Step 2: Build and test the image with "docker build --tag bulletinboard:1.0 ."

Step 3: Build and test the image with "docker build --tag bulletinboard:1.0 ."

Step 4: Run this command to start a container based off the pulled image: "docker run --publish 8000:8080 --detach --name bb bulletinboard:1.0"

Step 5: You can now visit the running containerized application on localhost:8000


Here are some basic Linux Commands to help you through this process:

mkdir - makes a new directory with a name you pass in within the current working directory you are in.

cd - allows you to change directory with the name of the chosen directory you pass in.

touch - touch allows you to create a new file with a specified file type with the given paramater you pass in.

vim - allows you to edit a filename you pass in on the flow within the terminal.

ls - lists all the files and directories within the current directory you are in.

rm - allows you to remove files and directories

git - utilize git commands to clone, pull, push, and interact with remote repositories.
