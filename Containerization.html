Docker, Linux, and Containerization



Containerization

A container is a form of software that packages up code and its dependencies to run and is an executable package of code essentially that runs an application on top of a given operating system. Containers bring a form of abstraction to the operating system level. Containers are great to use as they are light and fast applications that are quick to boot up. Docker is then used to package a given application in this form into containers. The Docker engine runs on top of the Host OS to then run containers on top of it to execute their own processes and utilize the resources of the host. You have a Dockerfile that contains all the commands to assemble the image, the image itself that is a template that is used to create the container, and then the container itself which is a running instance of a Docker Image that holds the package needed to run the given application you wish to execute.
